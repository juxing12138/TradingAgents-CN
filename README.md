## Overview
TradingAgents-CN evolves from the open-source TauricResearch/TradingAgents multi-agent trading research framework into a production-ready web product for Chinese-speaking individual professional traders.

The platform implements a layered decision-making workflow that mimics a real investment research team: analysts collect market, fundamental, and news data, followed by bull/bear debate, research synthesis, trader proposal, three-view risk review, and final ruling by a Portfolio Manager agent. It integrates multiple domestic and overseas data sources, enforces end-to-end data quality governance, and delivers auditable structured analysis reports via an intuitive Vue 3 single-page application.

> ⚠️ **Disclaimer**: This project is for technical research and educational purposes only. It does not constitute investment advice. All analysis outputs are generated by large language models and may contain errors. Users assume full responsibility for all investment decisions.

## Key Features
- 🧠 **Layered multi-agent decision pipeline**: 7 agent nodes with structured debate and risk governance for explainable conclusions
- 📊 **Multi-market data fusion**: Native support for A-share, Hong Kong, and US stocks with 10+ data providers and fallback mechanisms
- 🛡️ **Reliability governance**: Preflight data checks, immutable input snapshots, report evidence guards, and full task lineage audit
- 🌐 **Complete web product**: Single/batch analysis, task center, report management, watchlist, screening, paper trading, and system configuration
- ⚙️ **Flexible LLM support**: Compatible with OpenAI, DeepSeek, Qwen, Anthropic and other OpenAI-compatible model providers
- 🚀 **AI-accelerated development**: Built end-to-end with GPT-5.6 and OpenAI Codex as core development tools

## AI Development Tools: GPT-5.6 & OpenAI Codex
This project was developed with GPT-5.6 and OpenAI Codex as primary development accelerators. Rather than being only runtime inference models for end users, they served as core engineering tools throughout the entire development lifecycle, from architecture design to implementation, debugging, and documentation.

### GPT-5.6: Strategic Development Copilot
GPT-5.6 was used for high-complexity, high-abstraction engineering work that requires deep contextual understanding and cross-domain reasoning.

#### 1. Architecture Design & Technical Decision-Making
- Designed the three-layer architecture (agent core + backend service + frontend product) on top of the upstream research framework
- Evaluated and selected technology stacks: LangGraph for agent orchestration, MongoDB for persistence, Redis for progress tracking, and Vue 3 for the frontend
- Designed data governance strategies: canonical security identity system, data quality grading, provider fallback rules, and evidence traceability mechanisms
- Defined reliability guardrails: preflight checks, input snapshots, and report evidence guard logic

#### 2. Business Logic & System Design
- Mapped the full end-to-end analysis workflow: from symbol resolution → preflight → snapshot → agent execution → report generation → persistence
- Designed the task scheduling, retry lineage, and zombie task cleanup mechanisms
- Defined structured data schemas for research plans, trader proposals, and portfolio decisions
- Designed the dev/trial dual-environment isolation scheme and code synchronization workflow

#### 3. Debugging & Root Cause Analysis
- Diagnosed concurrency issues in async task execution and state consistency between memory and MongoDB
- Troubleshooted LangGraph state propagation bugs and conditional routing edge cases
- Analyzed multi-source data inconsistency issues across providers and designed normalization rules
- Identified performance bottlenecks in batch analysis and proposed optimization strategies

#### 4. Documentation & Knowledge Collation
- Generated the first draft of architecture documentation, API specifications, and this README
- Collated and structured the project cognitive baseline document for team knowledge transfer
- Generated test case designs for boundary conditions and failure scenarios
- Wrote inline code comments and docstrings for core modules

#### 1. Backend Business Code Generation

### Development Workflow Best Practices
We followed a structured workflow to maximize AI productivity while ensuring code quality:
1. **Design with GPT-5.6, implement with Codex**: GPT-5.6 defines interfaces, data structures and logic flow; Codex generates the actual implementation code
2. **Context-rich prompts**: Provide full project structure, existing code snippets, and architecture conventions to ensure generated code follows project standards
3. **Incremental integration**: Generate code in small, reviewable chunks rather than large monolithic blocks
4. **Human review for core logic**: All agent decision logic, data persistence code, and security-related code undergoes manual review before integration
5. **Iterative refinement**: Use error messages and test feedback to iteratively fix and improve generated code

## Runtime LLM Support
In addition to being development tools, GPT-5.6 and Codex are also natively supported as runtime inference models for the agent pipeline:
- **GPT-5.6**: Configurable as the deep reasoning model for Research Manager and Portfolio Manager nodes for highest quality analysis
- **Codex**: Powers the custom technical indicator and backtest script generation features in the stock detail page

Configuration is done via environment variables or the web-based system configuration panel:
```env
LLM_PROVIDER_OPENAI_ENABLED=true
OPENAI_API_KEY=your-api-key
OPENAI_BASE_URL=https://api.openai.com/v1
DEFAULT_DEEP_MODEL_NAME=gpt-5.6-sol
CODEX_ENABLED=true
```

## Quick Start
### Prerequisites
- Python 3.10+
- Node.js 18+
- MongoDB 6.0+
- Redis 7.0+

### Local Development Setup
1. Clone the repository
```bash
git clone https://github.com/your-username/TradingAgents-CN.git
cd TradingAgents-CN
```

2. Backend setup
```bash
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your MongoDB, Redis, and LLM API credentials
```

3. Frontend setup
```bash
cd frontend
npm install
```

4. Start development environment
```bash
# Back to project root
./scripts/devctl.sh dev start
```

5. Access the platform
- Web UI: http://localhost:5173
- API Docs: http://localhost:8000/docs
